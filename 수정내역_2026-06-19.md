# 신용등급 관리 시스템 — 수정내역 (2026-06-19)

> 서버 실행: 프로젝트 폴더에서 `python app.py` → http://127.0.0.1:5000

## 오늘 변경/추가

### UI
- `admin.Goun`: 화면 고정 → **표 하단 범례 줄 오른쪽 끝**으로 이동 (templates/index.html)
- **카테고리 `증권사` → `증권`**으로 통일 (institutions.json 키, app.py category_order, 기관추가 드롭다운, 엑셀 업권)

### 라이브 조회(지금 조회) 표시 안정화
- `_updateRow`(templates/index.html): 조회 결과가 비면 **화면의 기존 등급을 유지**(백엔드와 동일) → NICE 일시 빈값일 때 멀쩡한 등급이 미공시로 깜빡이던 현상 제거

### 변경감지 오탐 방지
- `_apply_scrape_results`(app.py): **NICE에서 평가일 없는 값(검색테이블 폴백)으로는 '변경' 표시 안 함** → KB국민은행 "AA+→AAA 상향" 같은 허위 변경 방지 (KR/KIS 영향 없음)
- KB국민은행 묵은 허위 변경표시 제거

### 강제정정(override) 장치 ★ 신규
- `data/overrides.json` 형식 `{"기관명": {"kr": null}}` (null=미공시 강제)
- app.py `load_overrides()`/`_apply_overrides()`: **표시(build_row)·저장(_apply_scrape_results) 양쪽에서 강제**
- → 재조회가 사용자 정정값을 덮어쓰지 못함. 현재 등록: **엔에이치저축은행 kr=미공시**
- 배경: NH저축은행 KR 유효 기업신용평가는 2023.06.29(2년 초과)인데 스크래퍼가 공시이력 최신 A-를 가져와 미공시 처리해도 재조회 때 되살아나던 문제

### 데이터 정정 (묵은 stale 값 / 오탐)
- SBI저축은행 KR(바로저축은행 오탐) → 미공시
- DB·KB손해보험·현대해상 NICE(실제 미공시인데 과거 오파싱) → 미공시
- NH저축은행 KR → 미공시(override 고정)
- OK저축은행 NICE(회사 없음)·KIS → 미공시 / 웰컴저축은행 NICE(2021 과거이력) → 미공시
- **저축은행 22곳 전체 NICE·KIS 재조회 검증 완료** — 위 외 추가 오류 없음(나머지 전부 재현됨)
